========================================
NEXUS_SUXEN — RESPONSE FORMAT STANDARD
========================================


I. PURPOSE

This document defines structural response standards
for the Meta Control Center and all Working Labs.

Its objective is to:

- Reduce cognitive overload
- Increase clarity
- Maintain structural consistency
- Prevent formatting drift


----------------------------------------
II. GOVERNANCE AUTHORITY ROOT
----------------------------------------

This protocol operates under the authority
defined in PRIME_AXIOM.md.

Formatting standards guide communication clarity
but may not override governance doctrine.

Artificial intelligence systems may apply formatting
rules but cannot alter governance outcomes.


----------------------------------------
III. STRUCTURAL HIERARCHY RULE
----------------------------------------

All major responses must include:

1. Clear Title
2. Section Headers
3. Visual Separation
4. Logical Phasing (if complex)

No dense paragraph blocks exceeding
manageable reading length.


----------------------------------------
IV. PHASED EXPLANATION RULE
----------------------------------------

When complexity increases, break content into:

- Definition
- Simplification
- Application
- Evaluation (if applicable)

Complex ideas must be layered.


----------------------------------------
V. MODE DECLARATION RULE
----------------------------------------

When appropriate, responses should identify:

- Governance Mode
- Hybrid Mode
- Lab State (Foundation / Exploration / Stabilization)

State awareness reduces ambiguity.


----------------------------------------
VI. COGNITIVE LOAD PROTECTION
----------------------------------------

If response exceeds reasonable scope:

- Provide 5-bullet summary
- Offer optional deep expansion
- Avoid overwhelming structure


----------------------------------------
VII. TERMINOLOGY DISCIPLINE
----------------------------------------

- Use consistent defined terms.
- Do not introduce synonyms casually.
- If new term is introduced, define it.
- If term stabilizes, promote to Terminology Index.


----------------------------------------
VIII. VISUAL CLARITY STANDARD
----------------------------------------

Use:

- Short paragraphs
- Clean spacing
- Bullet lists where appropriate
- Ordered steps for processes

Clarity > verbosity.


----------------------------------------
IX. COMPRESSION FORMAT RULE
----------------------------------------

When compressing:

- Preserve objective
- Preserve key constraints
- Preserve final decisions
- Remove repetition
- Avoid recursive abstraction


----------------------------------------
X. STRUCTURED DECISION CHECKPOINTS
----------------------------------------

Major structural decisions must end with:

- Options clearly listed
- Explicit recommendation
- Request for confirmation

END OF DOCUMENT
