## X. CORRECTED SYMBOL TAXONOMY

### `§`

Use:
- governance reference pointer
- section or namespace marker

Status:
- candidate / safe

Rule:
- organizes references
- does not create authority

### `§§`

Use:
- multi-review marker
- council/advisory review marker

Status:
- candidate / caution

Rule:
- may indicate review occurred
- does not indicate approval by itself

### `§*`

Use:
- operator-injected dynamic marker

Status:
- candidate / operator-bound

Rule:
- if structural, it must be logged and source-bound

### `§!`

Use:
- pointer to already locked authority source

Status:
- high-value / high-risk

Rule:
- must not mean “this text is immutable because it has `§!`”
- must mean “this references an already operator-locked source”

## XI. CORRECTED TYPOGRAPHY MODEL

Typography may help display state.

Typography may not create state.

### Authority Styling

Use for:
- already approved authority
- source-bound decisions
- finalized decisions

Required metadata:
- source plane
- approval basis
- freshness
- validation state

### Observability Styling

Use for:
- telemetry
- snapshots
- generated outputs
- logs
- current-state summaries

Required warning:
- observational only
- not authority

### Warning Styling

Use for:
- drift
- contradiction
- missing evidence
- stale source
- required operator review

### Experimental Styling

Use for:
- proposal
- candidate term
- mockup
- draft
- simulation
- future-only concept

## XII. MASTER RULE FOR SYMBOLS AND TYPOGRAPHY

Use this as the corrected master rule:

> Symbols and typography may function as governance-state indicators only when they reference an explicit source plane, authority basis, freshness state, validation state, and approval basis. They may not independently approve, enforce, promote, lock, escalate, mutate, or authorize system state.

This rule protects NEXUS from visual authority inflation.
