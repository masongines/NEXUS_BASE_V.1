# Report Template

Use this template for deep-dive outputs. Keep the report concise unless the operator asks for a full corpus-level assessment.

```markdown
# NEXUS Deep Dive Report: <subject>

## Status Block

- Status: observational / advisory / operator-review required
- Skill status: project-draft / auxiliary / not installed
- Scope: <roots, chats, and surfaces searched>
- Date: <date>
- Authority: no doctrine, implementation, promotion, or approval implied
- Confidence: high / medium / low

## Source Inventory

| Source | Class | Status | Evidence Count | Notes |
|---|---|---|---:|---|
| <path or conversation ID> | <source class> | read / indexed / gap | <count> | <notes> |

## Entity Map

| Entity | Observed role | Source class | Confidence | Notes |
|---|---|---|---|---|
| KAIROS/KAIROZ | <role> | <class> | <confidence> | <notes> |
| AKARA | <role> | <class> | <confidence> | <notes> |
| AEGIS | <role> | <class> | <confidence> | <notes> |
| NEXUS_SUXEN | <role> | <class> | <confidence> | <notes> |

## Origin And Timeline Lineage

State only what the searched sources support. Separate:

- directly evidenced sequence
- inferred sequence
- unresolved lineage claims

## Sovereign Council Findings

Describe:

- council purpose
- member/seat concepts
- KAIROS/KAIROZ relevance
- Hub Gate / Origin Coherence references
- voting, dissent, sandbox, or D&R rules

## AEGIS And NEXUS_SUXEN Findings

Describe:

- AEGIS role/status in current vs older sources
- NEXUS_SUXEN report lineage
- glossary/index terms
- local multi-LLM or document/information references

## Conflicts And Gaps

| Conflict or gap | Stronger source | Weaker source | Impact | Required action |
|---|---|---|---|---|
| <item> | <source> | <source> | <impact> | <operator/review action> |

## Assessment

Provide a bounded assessment. Do not promote findings. Label confidence.

## Recommendations

List concrete next steps:

- what to read next
- what requires OCR/PDF extraction
- what requires operator approval
- what should remain reference-only

## Approval Requirement

State whether approval is required for:

- promotion
- doctrine updates
- implementation
- install of this skill
- second-pass OCR/PDF extraction
```
