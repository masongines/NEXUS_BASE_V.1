NEXUS Base V1 — Context Export Generator Scope

Purpose:
Define the exact bounded scope of the context export generator lane.

Current Status:
- starter package stage
- no active context export generator code yet
- readiness packet exists
- lane classified as near-ready, not code-ready

Generator Target:
Context export generator only.

Current Implementation Objective:
Generate a read-only bounded context export artifact that helps orient the operator or assistant to the current Base V1 state without becoming governance truth, readiness authority, approval authority, or memory authority.

Included in First Pass:
- read approved Base V1 governance sources
- read approved operator planning/checkpoint sources
- read approved observational support sources
- assemble a bounded orientation artifact
- preserve stronger-vs-weaker source posture
- preserve source precedence explicitly
- write one context export output to the approved context export lane

Explicitly Excluded:
- doctrine modification
- registry modification
- manifest modification
- snapshot modification
- active standard modification
- support record modification
- automatic promotion
- readiness judgment generation
- recommendation generation
- runtime invocation
- historical trend engine behavior
- auto-repair or auto-fix behavior
- memory-style synthesis across non-approved roots
- PRE_DRAFT or LEGACY scanning by default

Output Rule:
The generated context export is an orientation and continuity surface only.
It does not become governance authority, doctrine identity source, readiness authority, approval authority, or operator-decision authority.

Current Ceiling:
- context export generator only
- no broader reporting engine implied
- no memory authority implied
- no approval authority implied
- no hidden connector or runtime behavior implied
