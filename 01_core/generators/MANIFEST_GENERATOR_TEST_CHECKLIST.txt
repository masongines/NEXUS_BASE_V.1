D. First-Pass Functional Checks
[ ] registry can be read

E. Interpretation Checks
[ ] manifest output states that doctrine identity comes from registry
[ ] manifest output does not claim doctrine authority
[ ] non-doctrinal artifacts remain non-doctrinal in output classification

F. Failure Handling
[ ] missing registry stops cleanly
[ ] malformed output is reported cleanly
[ ] no failure path mutates source files
[ ] no failure path expands into snapshot/context export generation

G. Gate Condition
[ ] operator approves first code draft only after this checklist is accepted
